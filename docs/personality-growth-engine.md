# UploadSoul 人格成长引擎 V1

## 产品目标

UploadSoul 不只解决“能想起哪条记忆”，还要逐步形成一个可生长的人格层。长期记忆负责事实和场景，人格层负责价值观、决策偏好、压力反应、敏感话题、表达习惯和情绪节奏。

## 当前 MVP 分层

1. 记忆采集层
   - 文本、语音、图片、访谈、聊天内容都会进入 `memory_fragments`。
   - 每条记忆保留时间、情绪、人物、地点、主题、重要性、可信度和向量。

2. 记忆检索层
   - 使用 Voyage 向量检索召回相关记忆。
   - 对话界面展示回答来源，让用户知道“这句话来自哪条记忆”。

3. 人格成长层
   - V1 存在 `profiles.metadata.personality_model`，避免新增强制 SQL。
   - 从记忆片段和用户反馈中蒸馏：
     - 核心价值观
     - 决策偏好
     - 压力反应模式
     - 敏感话题
     - 表达习惯
     - 情绪反应模式
     - 声音情绪控制建议
     - 主动触发规则

4. 情绪与声音层
   - 每轮聊天先分析当前输入和被唤起记忆的情绪状态。
   - 回答会根据情绪状态和人格模型调整语言节奏。
   - 语音侧保留疲惫、兴奋、犹豫、敏感话题停顿等控制字段，后续可传给更强的 TTS 或实时语音引擎。

5. 主动触发层
   - 当前 V1 已生成可见建议：
     - 纪念日提醒
     - 沉默期问候
     - 高情绪记忆触发
     - 触景生情线索
     - 碎金记忆
   - 后续可接 Vercel Cron 或 Supabase Edge Functions 定时执行。

## 已接入接口

- `GET /api/personality/model?profile_id=...`
  - 返回当前人格模型、主动触发建议和统计。

- `POST /api/personality/model`
  - 根据当前记忆和反馈重新蒸馏人格模型。

人格模型已经注入：

- `/api/memory-chat`
- `/api/test-chat`
- `/api/test-chat-stream`

## 后续升级路线

1. 主动触发任务
   - 每天检测纪念日。
   - 检测用户沉默天数。
   - 检测近期高压或低落情绪。
   - 将合适的触发建议写入通知队列。

2. 人格版本管理
   - 从 metadata 升级为独立 `personality_snapshots` 表。
   - 保留每次蒸馏版本，支持回滚和比较。

3. 语音情绪控制
   - 将 `voice_emotion_profile` 映射到 TTS 参数。
   - 对敏感话题加入短停顿、慢速、低能量表达。
   - 对兴奋话题提高语速和亮度。

4. 多模态触景生情
   - 用户上传照片、视频、地点或物品时，先做多模态识别。
   - 将识别结果与记忆主题和地点匹配，触发相似回忆。

5. 算力到位后的模型训练
   - `memory_fragments` 提供事实数据。
   - `personality_model` 提供稳定人格标签。
   - `conversations` 和 `memory_feedback` 提供偏好纠偏数据。
   - 三者一起组成未来微调、偏好学习和个性化语音模型的数据底座。
